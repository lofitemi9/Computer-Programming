1. Fluid Typography & Clip-Path

We're using the viewport width unit (vw) to scale the heading font size.

Fluid Scaling

CSS Units: rem, vw, and the Clip-Path Trick

Relative Units:rem is great for text. vw (viewport width) is used above to make the heading scale directly with the browser width.

Clip-Path: This advanced property creates complex shapes. Here, it creates a custom, sharp diagonal edge at the bottom of the dark header section.

/* Using vw for fluid scaling on the H3 */ .fluid-heading { font-size: 8vw;} /* Defines the sharp, custom shape (Clip-Path) */ .clip-path-hero { clip-path: polygon(0 0, 100% 0, 100% 90%, 0% 100%);}

3. CSS Grid for Responsive 2D Layouts

This layout is built **Mobile-First** (1 column) and switches to 3 columns using an explicit **Media Query** at 768px.

Header / Wide Content

On desktop (> 768px), this item uses grid-column: span 2 to cover two columns.

Sidebar 1

One column wide on all screen sizes.

Sidebar 2

One column wide on all screen sizes.

Footer / Full Width

This item also spans two columns on desktop, creating a balanced bottom row with the remaining column.

The Responsive Grid Code

/* Mobile First Default: All items stack (1 column) */ .responsive-grid { display: grid; grid-template-columns: 1fr; } /* Media Query for Tablet/Desktop (>= 768px) */ @media (min-width: 768px) { .responsive-grid { grid-template-columns: repeat(3, 1fr); /* 3 equal columns */ } .col-span-2 { grid-column: span 2 / span 2; } }

4. Advanced: Art Direction with <picture>

The <picture> element allows us to serve **different images** for different screen sizes. This is crucial when simple scaling is not enough (e.g., cropping a wide banner to a zoomed-in detail on mobile).

The Code in Action

/* The browser uses the first matching source, falling back to */ <picture> <source media="(min-width: 768px)" srcset="desktop-wide.jpg"> /* The mobile default image */ <img src="mobile-crop.jpg" alt="Art Directed Image"> </picture>
